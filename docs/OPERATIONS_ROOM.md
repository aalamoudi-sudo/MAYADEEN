# KAG Operations Room

## الهدف

تحويل النظام من لوحة متابعة إلى غرفة عمليات يومية تدير التنفيذ، التصعيد، وجاهزية الافتتاح بقرار واضح:

- Go
- Conditional Go
- No-Go

## التبويبات المطلوبة في غرفة العمليات

### 1. Command View

الشاشة الرئيسية لغرفة العمليات.

تعرض:
- حالة المشروع العامة.
- قرار Go/No-Go الحالي.
- أهم 10 عوائق تشغيلية.
- مؤشرات المخاطر والقرارات والاعتمادات.
- آخر تحديثات الفرق.

### 2. Readiness Checklist

قائمة جاهزية الإطلاق.

تعرض:
- البنود الحرجة قبل الافتتاح.
- مالك كل بند.
- الحالة: Ready, At Risk, Not Ready.
- الدليل المطلوب للإغلاق.
- تاريخ آخر تحديث.

### 3. Incident Log

سجل الحوادث التشغيلية.

يستخدم في يوم التشغيل أو التجارب الميدانية.

يعرض:
- رقم الحادث.
- الأولوية.
- المالك.
- وقت الفتح.
- وقت الحل.
- الحالة.
- القرار أو الإجراء المتخذ.

### 4. Team Status

حالة الفرق.

يعرض:
- كل مسار عمل.
- آخر تحديث.
- العوائق.
- الاحتياج من الإدارة.
- جاهزية الفريق.

### 5. Go/No-Go Board

مجلس قرار الإطلاق.

يعرض:
- نتيجة كل محور جاهزية.
- الشروط غير المغلقة.
- قرار اللجنة.
- المالك التنفيذي.
- توقيت المراجعة القادمة.

### 6. War Room Timeline

خط زمني للـ 72 ساعة الحرجة.

يعرض:
- T-72 / T-48 / T-24 / T-12 / T-6 / Launch.
- نقاط الفحص.
- القرارات المطلوبة.
- الحوادث المفتوحة.

### 7. After Action Review

مراجعة ما بعد التشغيل.

تعرض:
- ما حدث.
- ما تم حله.
- الدروس المستفادة.
- التحسينات المطلوبة.

## التقنيات التي يجب ربطها

### Slack أو Microsoft Teams

للتنبيهات والتصعيد:
- Critical task overdue.
- Incident opened.
- Decision overdue.
- Approval blocking launch.

### Google Workspace أو Microsoft 365

للهوية والمستندات:
- Single Sign-On.
- مستندات الاعتماد.
- محاضر الاجتماعات.

### Supabase

للتنفيذ السريع:
- PostgreSQL.
- Auth.
- Row Level Security.
- Realtime channels.

### Vercel

لاستضافة الواجهة:
- Preview deployments.
- Production deployment.
- Environment secrets.

### Power BI أو Looker Studio

للتقارير التنفيذية إذا رغبت الإدارة بتقارير خارج النظام.

### GitHub

لإدارة الكود:
- Issues.
- Pull requests.
- Security scanning.
- Release tracking.

## قواعد التشغيل

1. لا يوجد Go إذا توجد مخاطر حرجة مفتوحة.
2. لا يوجد Go إذا توجد قرارات عاجلة غير محسومة.
3. لا يوجد Go إذا توجد مهام حرجة متأخرة.
4. Conditional Go مسموح فقط إذا كانت كل البنود الحرجة لها مالك وخطة إغلاق وتاريخ واضح.
5. كل حادث تشغيلي يجب أن يملك صاحب قرار واحد.
6. كل تحديث تشغيلي يجب أن يحفظ في audit log.

## مسار العمل اليومي

1. تحديث الفرق قبل 10:00 صباحاً.
2. اجتماع غرفة العمليات 10:30 صباحاً.
3. إغلاق القرارات العاجلة قبل 2:00 مساءً.
4. تحديث تقرير الجاهزية قبل 5:00 مساءً.
5. إرسال Executive Brief بنهاية اليوم.

## تعريف الجاهزية

المشروع جاهز عندما تتحقق الشروط التالية:

- لا توجد مهام حرجة متأخرة.
- لا توجد مخاطر حرجة مفتوحة بدون خطة معالجة.
- لا توجد قرارات عاجلة معلقة.
- الاعتمادات الأساسية مغلقة أو مقبولة بشرط واضح.
- كل الفرق حدثت حالتها خلال آخر 24 ساعة.
- توجد خطة تشغيل يوم الإطلاق.
- توجد خطة رجوع أو بدائل للحالات الحرجة.
