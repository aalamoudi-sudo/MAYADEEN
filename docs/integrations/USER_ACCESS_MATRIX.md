# مصفوفة المستخدمين والصلاحيات

## المبدأ

الـ PMO لديه كل الصلاحيات:

- مشاهدة كل الصفحات.
- تعديل السجلات التشغيلية.
- إدارة الاعتمادات والتصعيد.
- إدارة المستخدمين.
- إصدار التقارير والرسائل.

باقي المستخدمين يحصلون على صلاحيات حسب دورهم في هيكلة ميادين.

## شيت الصلاحيات

ينشئ Apps Script شيت باسم:

`User Access Matrix`

## الأعمدة المهمة

| العمود | الوصف |
| --- | --- |
| `username` | اسم المستخدم |
| `temporary_password` | كلمة مرور مؤقتة للتشغيل الأول |
| `password_hash` | اختياري لاحقاً عند التحويل لتخزين مشفر |
| `display_name` | اسم الشخص |
| `email` | البريد |
| `role` | الدور |
| `access_level` | مستوى الصلاحية |
| `path_scope` | نطاق المسار |
| `allowed_pages` | الصفحات المسموحة، أو `*` لكل الصفحات |
| `can_approve` | صلاحية الاعتماد |
| `can_escalate` | صلاحية التصعيد |
| `can_manage_users` | إدارة المستخدمين |
| `status` | active / inactive |
| `must_change_password` | TRUE عند كلمة مرور مؤقتة |

## الأدوار

- `PMO`: صلاحية كاملة `full` و `allowed_pages = *`.
- `مشرف عام داخلي`: عرض تنفيذي وتصعيد.
- `مدير المشروع`: تشغيل واعتمادات وتنفيذ.
- `مدير الجودة والمخاطر`: مخاطر، جودة، واعتمادات.
- `مدير الحدث`: تشغيل ميداني ومسارات الحدث.
- `منسق المشروع`: تحديثات، محاضر، توزيع مهام، وتذكيرات.
- `مدير الحساب والعلاقات الحكومية`: بوابات خارجية وتصاريح.
- `قائد مسار`: مهامه، تكليفاته، ملفاته، واجتماعاته.
